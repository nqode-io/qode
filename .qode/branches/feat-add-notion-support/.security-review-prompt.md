## Security Checklist

Review each applicable area for the tech stacks above:

### Injection
- SQL injection (parameterised queries used?)
- NoSQL injection
- Command injection (user input in shell commands?)
- LDAP injection
- Template injection

### Authentication & Authorisation
- Authentication bypass possibilities
- Missing authorisation checks
- Privilege escalation paths
- Insecure direct object references (IDOR)
- JWT/session token handling

### Data Exposure
- Sensitive data in logs
- PII/secrets in error messages
- Over-fetching in API responses
- Insecure data storage

### Input Validation
- All user inputs validated and sanitised
- File upload restrictions
- Integer overflow / boundary checks
- Regex denial of service (ReDoS)

### Cryptography
- Weak algorithms (MD5, SHA1, DES)
- Hardcoded secrets or API keys
- Insecure random number generation
- Improper certificate validation

### Frontend-Specific (React / Angular)
- XSS via dangerouslySetInnerHTML or innerHTML
- CSP violations
- Sensitive data in localStorage/sessionStorage
- Open redirects

### API Security
- Rate limiting present?
- CORS misconfiguration
- Mass assignment vulnerabilities
- Insecure deserialization

### Dependency Issues
- New dependencies added? Are they well-maintained?
- Known CVEs in new packages?
