# Technical Specification: Fix `qode plan refine` Scoring

*Branch: bug-plan-refine-scoring-not-done*

---

## 1. Feature Overview

`qode plan refine` is supposed to iterate a requirements analysis through a worker/judge two-pass scoring loop until the analysis reaches 25/25. After each iteration the score should be printed, persisted in the filename, and embedded in the file header so `qode plan status` can display progress.

Three bugs and one skill gap cause every score to be recorded as 0/25: the judge prompt is dispatched with a placeholder instead of the real analysis; iteration files are written before the judge runs; and `LatestScore()` conflates iteration number with score in a single variable. The IDE slash commands (Claude Code and Cursor) never invoke the judge at all.

This spec covers the changes required to make scoring work correctly in both the terminal dispatch path and the IDE slash command path.

**Success criteria:**
- `qode plan refine` (dispatch mode) prints the real judge score and persists it correctly
- `qode plan status` shows correct per-iteration scores
- `/qode-plan-refine` in Claude Code and Cursor completes the full worker+judge loop in a single invocation
- All existing tests pass; new tests are added for the fixed/new functions

## 3. Architecture & Design

### Component interaction (dispatch mode, before fix)

```
runPlanRefine
  └─ BuildRefinePromptWithOutput      ← builds judge prompt with PLACEHOLDER
  └─ SaveIterationFiles
  └─ refineDispatch
       ├─ d.Run(workerPrompt)          → worker output
       ├─ read savedAnalysis from file
       ├─ ParseIterationFromOutput     ← saves score=0 to disk (TOO EARLY)
       ├─ d.Run(out.JudgePrompt)       ← judge sees PLACEHOLDER, returns 0
       └─ result overwritten in memory (files on disk NOT updated)
```

### Component interaction (dispatch mode, after fix)

```
runPlanRefine
  └─ BuildRefinePromptWithOutput      ← judge prompt still has placeholder (unchanged)
  └─ SaveIterationFiles               ← .refine-judge-prompt.md still has placeholder (for --prompt-only)
  └─ refineDispatch(engine)
       ├─ d.Run(workerPrompt)          → worker output
       ├─ read savedAnalysis from file
       ├─ scoring.NewEngine.BuildJudgePrompt(savedAnalysis)  ← FRESH prompt with real content
       ├─ d.Run(freshJudge)            → real judge score
       ├─ scoring.ParseScore(judgeOutput)
       └─ SaveIterationResult(result)  ← saves correct score to disk (AFTER judge)
```

## 4. API / Interface Contracts

### New function: `plan.SaveIterationResult`

```go
// SaveIterationResult writes iteration files using a pre-computed score result.
// Use this instead of ParseIterationFromOutput when the score is known from
// judge output rather than parsed from the analysis text.
func SaveIterationResult(root, branch string, iteration int, analysisText string, result scoring.Result) error
```

**Writes:**
- `{branchDir}/refined-analysis-{iteration}-score-{result.TotalScore}.md` — archived copy of analysis
- `{branchDir}/refined-analysis.md` — canonical latest file with header `<!-- qode:iteration=N score=S/M -->`

**Returns:** first write error encountered, or nil.

**Does not replace `ParseIterationFromOutput`** — existing function is retained unchanged.

### Modified function: `LatestScore`

```go
// Before (buggy):
func (c *Context) LatestScore() int {
    best := 0
    for _, it := range c.Iterations {
        if it.Number > best {
            best = it.Score
        }
    }
    return best
}

// After:
func (c *Context) LatestScore() int {
    latestNum := 0
    latestScore := 0
    for _, it := range c.Iterations {
        if it.Number > latestNum {
            latestNum = it.Number
            latestScore = it.Score
        }
    }
    return latestScore
}
```

### Modified: slash command text (Claude Code + Cursor)

The `"qode-plan-refine"` entry in `claudeSlashCommands()` (`claudecode.go`) and `slashCommands()` (`cursor.go`) is updated to include the judge-execution steps after the worker analysis is saved. The `.claude/commands/qode-plan-refine.md` static file reflects the same content as `claudeSlashCommands()`.

New behaviour described in the command text:
1. Run `qode plan refine --prompt-only`
2. Execute worker prompt → save analysis to `refined-analysis.md`
3. Read `.refine-judge-prompt.md`; substitute the placeholder line with the content of `refined-analysis.md`
4. Execute the modified judge prompt
5. Parse `**Total Score:** N/25` from judge output
6. Detect iteration number from existing `<!-- qode:iteration=N -->` header (default 1)
7. Rewrite `refined-analysis.md` with `<!-- qode:iteration=N score=S/25 -->` header
8. Write `refined-analysis-{N}-score-{S}.md`
9. Report score; suggest `qode plan spec` if S ≥ 25, else suggest re-running the command
