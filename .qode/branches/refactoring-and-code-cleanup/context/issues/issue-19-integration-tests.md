# Issue #19: Integration Test Layer for CLI Commands

## Summary

Unit tests cover individual packages well (config, context, scoring, plan, review, workflow), but no test exercises the full command pipeline — from CLI invocation through config loading, git context, branch context loading, and template rendering to stdout. A regression in template rendering, context loading, or prompt assembly can slip through undetected. A golden-file integration test suite would catch these regressions at the pipeline level.

The prompts generated by CLI commands are the direct output consumed by LLMs — a silently malformed prompt is a high-impact bug.

### What is NOT tested

No test exercises the full `qode plan refine` flow:

1. Root resolution (`config.FindRoot()`)
2. Config loading (`config.Load()`)
3. Git context (`git.CurrentBranch()`)
4. Branch context loading (`context.Load()`)
5. Template rendering (`prompt.NewEngine()` + `engine.Render()`)
6. Stdout prompt output

Steps 2, 4, and 5 have unit tests — but no test runs all six steps together with realistic fixtures.

**Also untested as a whole:**

- Multi-step workflows (`plan refine` → `plan judge` → `plan spec`)
- Custom scoring rubrics propagating through the full pipeline
- Branch names with special characters (sanitization in file paths)
- Guard checks combined with real file state (not mocked)
- Stdout output content and formatting

## Proposed Fix

Create `internal/cli/integration_test.go` with a golden-file approach.

### Project setup helper

```go
// setupProject creates a temp git repo with the standard .qode/ layout.
func setupProject(t *testing.T, branch string, ticket, notes string) string {
    t.Helper()
    root := t.TempDir()

    // init git
    exec.Command("git", "-C", root, "init").Run()
    exec.Command("git", "-C", root, "checkout", "-b", branch).Run()

    // write qode.yaml
    os.WriteFile(filepath.Join(root, "qode.yaml"), []byte("version: 1\n"), 0644)

    // create branch context
    branchDir := filepath.Join(root, ".qode", "branches", git.SanitizeBranchName(branch), "context")
    os.MkdirAll(branchDir, 0755)
    if ticket != "" {
        os.WriteFile(filepath.Join(branchDir, "ticket.md"), []byte(ticket), 0644)
    }
    if notes != "" {
        os.WriteFile(filepath.Join(branchDir, "notes.md"), []byte(notes), 0644)
    }
    return root
}
```

### Command execution helper

```go
// runCommand executes a cobra command and returns captured stdout.
func runCommand(t *testing.T, root string, args ...string) (string, error) {
    t.Helper()
    // temporarily change to root so resolveRoot() works
    origDir, _ := os.Getwd()
    os.Chdir(root)
    defer os.Chdir(origDir)

    var buf bytes.Buffer
    cmd := NewRootCmd()
    cmd.SetOut(&buf)
    cmd.SetArgs(args)
    err := cmd.Execute()
    return buf.String(), err
}
```

### Golden file comparison

```go
func assertGolden(t *testing.T, got, name string) {
    t.Helper()
    goldenPath := filepath.Join("testdata", name+".golden")
    if *update { // -update flag regenerates goldens
        os.WriteFile(goldenPath, []byte(got), 0644)
        return
    }
    expected, err := os.ReadFile(goldenPath)
    if err != nil {
        t.Fatalf("missing golden file %s (run with -update to create)", goldenPath)
    }
    if got != string(expected) {
        t.Errorf("output mismatch for %s:\n%s", name, diff(string(expected), got))
    }
}
```

### Test cases

```go
func TestIntegration_PlanRefine_DefaultRubric(t *testing.T) {
    root := setupProject(t, "feat/my-feature",
        "# Implement OAuth login\n...",
        "Use PKCE flow for mobile clients.",
    )
    out, err := runCommand(t, root, "plan", "refine")
    if err != nil {
        t.Fatalf("plan refine failed: %v", err)
    }
    assertGolden(t, out, "plan_refine_default")
}

func TestIntegration_PlanRefine_MissingTicket(t *testing.T) {
    root := setupProject(t, "feat/no-ticket", "", "")
    _, err := runCommand(t, root, "plan", "refine")
    // should succeed (ticket is optional) but produce a prompt without ticket section
    if err != nil {
        t.Fatalf("unexpected error: %v", err)
    }
}

func TestIntegration_PlanSpec_GuardBlocked(t *testing.T) {
    root := setupProject(t, "feat/no-analysis", "# Some ticket", "")
    // no refined-analysis.md exists → guard should block
    _, err := runCommand(t, root, "plan", "spec")
    if err == nil {
        t.Error("expected guard error, got nil")
    }
}
```

## Existing test infrastructure to reuse

`internal/cli/plan_test.go` already has:

- `setupPlanTestRoot()` helper (creates temp dir + branch context)
- `captureStdout()` helper (os.Pipe-based, can be replaced with io.Writer after issue #17)

The integration tests build on these patterns with real git init and end-to-end command execution.

## Impact

**Regressions caught:**

- Template rendering changes (field name renames, missing template variables)
- Context loading bugs (missing sections in assembled prompt)
- Config merging issues (custom rubric not propagating)
- Guard check changes (blocking conditions, error messages)
- Output formatting and section ordering

**What unit tests miss that integration tests catch:**

- The full `resolveRoot → config → git → context → engine → render` chain
- Real file I/O interaction between steps
- Branch name sanitization in actual file paths
- Template variable population from real context files

**Scope:** Integration tests live alongside unit tests in `internal/cli/`; CI runs them as part of `go test ./...`. Test execution time increases modestly (each test creates a temp dir and runs in-process cobra execution — no subprocess overhead if issue #17 is addressed first).
