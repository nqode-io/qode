## 1. Feature Overview

The `/qode-plan-refine` slash command is intended to support multi-round iterative refinement, where each invocation advances the iteration counter (Iteration 1 → Iteration 2 → …). Currently, the command is not re-entrant: every invocation after the first generates an "Iteration 1" prompt instead of the correct next iteration number. This breaks the in-IDE refinement workflow and forces users to context-switch to the terminal.

The root cause is that `context.Load()` tracks iterations exclusively via glob-matched numbered files (`refined-analysis-N-score-S.md`), which are only created by the judge-scoring path (`qode plan refine` without `--prompt-only`). The slash command writes directly to `refined-analysis.md` with an HTML comment header encoding the iteration number, but this file is never read back by `context.Load()`.

The fix adds a `parseIterationFromAnalysis()` helper that reads the HTML comment header from `refined-analysis.md` and uses it as a fallback iteration source in `context.Load()`. This is a minimal, self-contained change with no schema or interface changes.

**Success criteria:** Two successive `/qode-plan-refine` invocations produce "Iteration 1" then "Iteration 2". `qode check` passes.
